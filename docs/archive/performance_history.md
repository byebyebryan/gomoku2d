# Performance History

Historical summary of major performance/search experiments. Detailed dated logs
were intentionally removed from the active working note so current docs stay
readable.

## Search-Bot Tuning

Depth, candidate caps, tactical ordering, and pattern eval all mattered, but
direct strength tuning reached diminishing returns quickly under browser-scale
compute budgets. The durable result was a measured anchor set and explicit
search specs, not one magic knob.

## Corridor Search In Live Bot Search

Several forms were tested:

- corridor bot by itself;
- corridor portals during search;
- leaf corridor extension;
- after-search corridor proof.

The portal/leaf forms were too expensive and did not justify default use. The
after-search corridor proof path remains available as a narrow lab axis, but
replay analysis is the main successful use of corridor search.

## Rolling Frontier / Threat View

Rolling threat facts became worthwhile because more of the system depended on
threat queries: search safety, ordering, pattern eval, hints, replay analysis,
and Renju legality interactions.

The important design decision was to add rolling alongside scan-backed fallback
and shadow validation before relying on it as the default.

## Renju Legality

The old shape shortcut was replaced with a recursive legality oracle because
Renju double-three/double-four depends on real continuations, not raw shapes.
Performance was recovered through cheap necessary-condition guards while
preserving the exact oracle as the authority.

## Report Pipeline

Published reports moved from static HTML generated by Rust to compact JSON plus
web-rendered pages. Full diagnostic telemetry belongs in ignored scratch output;
published data should keep only the data needed for the product-facing report.
